# Athena v4 Preserved Semantics

This file is the behavior checklist for the v4 rewrite. Each item must map to one or more regression tests. A semantic without a test is a spec bug.

Do not edit this file casually. Any PR that changes behavior must update the matching tests in the same commit.

## Tiered autonomy

- Tier 0 actions run silently for risk-free reads.
- Tier 1 actions execute and write an `action_taken` audit record.
- Tier 2 actions block on explicit UI approval and record the grant in `audit_log`.

## Skill trust chain

- Crystallized skills are saved as unverified.
- Loading an unverified skill is Tier 2.
- Chain-loading an unverified skill from a verified skill is still Tier 2.
- User approval promotes a skill atomically.
- Background agents can never pass a Tier 2 gate, including in AUTO_APPROVE mode.

## Crystallization

- A `/task` with 4 or more tool calls triggers the post-task crystallization hook.
- `load_skill` calls are filtered out of the trace before summarization.
- The crystallizer saves or updates repeatable skills.
- A skill version with more than 70 percent success rate is never overwritten; a new version is written beside it.
- Rollback is manual through `skill_rollback` or the Skills view.

## CORAL peer learning

- CORAL is versioned, append-only, and platform-filtered.
- Agents pull CORAL updates only at turn boundaries, never mid-turn.
- CORAL is persisted in `coral_log`.

## Watchers

The 9 v3 monitors carry forward with their exact triggers:

- `disk_low`: every 5 minutes; critical below 5GB, low below 15GB, or more than 2GB drop.
- `kp41`: every 10 minutes; Windows Kernel-Power 41.
- `temp_high`: every 3 minutes; Linux CPU temp at or above 80C high, 90C critical.
- `net_change`: every 2 minutes; interfaces added or removed.
- `ram_pressure`: every 3 minutes; below 500MB free low, below 300MB critical.
- `cpu_spike`: every 2 minutes; above 95 percent twice consecutively.
- `battery_drain`: every 5 minutes; below 10 percent or more than 5 percent drop in 3 minutes.
- `login_failures`: every 5 minutes; more than 5 failed attempts in the window.
- `pending_reboot`: every 60 minutes.

Watcher semantics:

- Related alerts inside 5 minutes merge into one combined event.
- Alerts raised mid-turn are queued and drained at the next safe turn boundary.
- Critical alerts checkpoint current task state so interrupted work is resumable.
- Alert lifecycle is an FSM: `created -> shown -> acked | resolved | escalated | expired`.
- Alert state is persisted in `alerts` and `alert_events`.

## Memory

- Jaccard-similarity GC merges near-duplicate entries.
- Contradiction detection flags conflicts between new entries and validated existing entries for user resolution.
- Age decay down-weights recall ranking but does not silently delete memories.
- Instinct confidence is tracked as `[conf][domain][seen]`.
- Instinct scoring considers the last 60 session files.
- Promotion requires confidence >= 85 and presence in 3 or more distinct sessions.
- Confidence is reinforced by a cheap-model attribution pass during session summary.
- Stale instincts retire below confidence 40.
- Machine-scoped instincts remain machine-scoped where applicable.
- Every instinct change writes an `instinct_events` row.
- Prohibited patterns are enforced at memory-write time and tool-execution time.
- Hybrid recall tries vector similarity first and BM25 keyword fallback when embeddings are unavailable.
- Recall must work without network and without API keys.

## Machines

- Fingerprint is a SHA-256 hash over stable host identifiers.
- Raw identifiers such as MACs and usernames are not stored in plaintext.
- Each boot records a snapshot.
- Longitudinal diffs across visits are surfaced in the Watchers/System experience.

## Compression

- Rolling compression happens around a 40-message window.
- Tool results are summarized into compressed context, not dropped.
- Output caps are per tool, not a single global cap.

## Errors

- Runtime errors use `AthenaError { code, source, severity, message, hint }`.
- Bare `.catch(() => {})` is banned by lint.
- The sanctioned swallow path is `ctx.errors.swallow()`, which writes `errors` and emits `error_detail`.
- The UI exposes an error drawer.

## Risk and approvals

- Risk verdicts are deterministic only.
- Policy is data, not code.
- Matching is case-insensitive and quote-normalization-aware.
- Unparseable structure, decode failure, or recursion deeper than 3 fails closed to Tier 2.
- Command substitution is minimum Tier 2.
- Redirects to system paths are Tier 2.
- Elevation through sudo, doas, runas, or equivalent raises risk.
- Unknown binaries have at least Tier 1.
- Every command caught by v3 `classifyRisk()` must classify at least as high in v4.
- The bypass corpus grows monotonically and covers POSIX and Windows.

## AUTO_APPROVE

- AUTO_APPROVE applies only to the interactive user's own turns.
- AUTO_APPROVE never applies to background agents.
- AUTO_APPROVE never permits unverified skill loading without a gate.
- Auto-approved actions still write audit rows flagged `auto_approved`.
- The UI shows a persistent warning banner while AUTO_APPROVE is enabled.

## Provider failover

- Two consecutive 429, 401, or 403 errors block a provider for 15 minutes.
- Provider state is persisted in `provider_health`.
- Failover emits a visible event; it is never silent.

## Regression authority

- The 61 v3 acceptance specs are ported against v4 events and observable behavior.
- Injection, trust-chain, watcher, memory, compression, and migration semantics are CI-blocking.
